# 自定义执行计划配置

## ✅ 当前配置

**执行频率**：每天 8:00-22:00（北京时间）每 120 分钟（每 2 小时）运行一次

**Cron 表达式**：`0 0-14/2 * * *`

## 📅 执行时间表

### UTC 时间（GitHub Actions 使用）
- 00:00, 02:00, 04:00, 06:00, 08:00, 10:00, 12:00, 14:00

### 北京时间（实际推送时间）
- **08:00** - 第一次运行
- **10:00** - 第二次运行
- **12:00** - 第三次运行
- **14:00** - 第四次运行
- **16:00** - 第五次运行
- **18:00** - 第六次运行
- **20:00** - 第七次运行
- **22:00** - 最后一次运行

**每天共运行 8 次**

## ⚠️ 重要提示

1. **时间偏差**：
   - GitHub Actions 的定时任务可能有 ±20 分钟偏差
   - 实际执行时间可能略有不同

2. **UTC 时间**：
   - GitHub Actions 使用 UTC 时间
   - 北京时间 = UTC 时间 + 8 小时

3. **推送模式**：
   - 当前推送模式：`daily`（当日汇总）
   - 如需修改，编辑 `config/config.yaml`

## 🔄 如何修改

如果需要调整执行频率，编辑 `.github/workflows/crawler.yml`：

```yaml
- cron: "0 0-14/2 * * *" # 当前配置
```

**其他示例**：
- 每 1 小时：`0 0-14 * * *`
- 每 30 分钟：`*/30 0-14 * * *`
- 每 3 小时：`0 0-14/3 * * *`

## 📝 下一步

1. **提交更改**：
   ```bash
   git add .github/workflows/crawler.yml
   git commit -m "配置执行频率：每天 8:00-22:00 每 2 小时运行一次"
   git push origin master
   ```

2. **测试配置**：
   - 在 GitHub Actions 中手动触发一次
   - 验证配置是否正确

3. **等待自动运行**：
   - 系统会在每天 8:00 开始第一次运行
   - 之后每 2 小时运行一次，直到 22:00
